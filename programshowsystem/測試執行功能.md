# 測試 Python 執行功能

## 快速測試步驟

### 1. 測試簡單程式

上傳以下測試程式碼：

```python
print("Hello, World!")
print("測試成功！")
```

**預期結果**：應該顯示兩行輸出

### 2. 測試計算

```python
x = 10
y = 20
result = x + y
print(f"{x} + {y} = {result}")
```

**預期結果**：應該顯示 "10 + 20 = 30"

### 3. 測試函數

```python
def greet(name):
    return f"Hello, {name}!"

message = greet("Python")
print(message)
```

**預期結果**：應該顯示 "Hello, Python!"

## 檢查執行狀態

### 在瀏覽器控制台檢查

按 **F12** 打開控制台，執行以下命令：

```javascript
// 檢查 Pyodide 是否載入

```

## 常見問題

### 問題：點擊執行沒有反應

**檢查**：
1. 按 F12 打開控制台
2. 查看是否有錯誤訊息
3. 確認 Pyodide 是否已載入

### 問題：執行後沒有輸出

**可能原因**：
1. 程式碼沒有 `print()` 語句
2. 輸出捕獲系統未正確初始化
3. 程式碼執行出錯

**解決方案**：
1. 確認程式碼中有 `print()` 語句
2. 查看控制台的錯誤訊息
3. 嘗試執行簡單的測試程式

### 問題：顯示「執行完成，無輸出內容」

**這表示**：
- 程式執行成功
- 但沒有輸出內容

**解決方案**：
- 確認程式碼中有 `print()` 語句
- 檢查程式邏輯是否正確

## 調試技巧

### 1. 查看控制台日誌

執行時，控制台會顯示：
- "開始執行程式碼..."
- "程式碼執行完成"
- "輸出獲取完成"
- "結果顯示完成"

如果缺少某些日誌，表示在該步驟出現問題。

### 2. 測試輸出捕獲

在控制台執行：

```javascript
pyodide.runPython(`
import sys
from io import StringIO

class OutputCapture:
    def __init__(self):
        self.buffer = StringIO()
    def write(self, s):
        if s is not None:
            self.buffer.write(str(s))
    def flush(self):
        pass
    def getvalue(self):
        return self.buffer.getvalue()

stdout_capture = OutputCapture()
sys.stdout = stdout_capture
print("測試輸出")
sys.stdout.getvalue()
`)
```

應該返回 "測試輸出\n"

---

**提示**：如果問題仍然存在，請查看瀏覽器控制台的詳細錯誤訊息。
