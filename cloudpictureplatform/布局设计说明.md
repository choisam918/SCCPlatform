# 网上云展览系统 - 布局设计说明

## 一、整体布局架构

系统采用**两大核心模块**的清晰布局结构：

```
┌─────────────────────────────────────────┐
│          主页 (index.html)              │
│                                         │
│    ┌────────────┐  ┌────────────┐     │
│    │  上传照片   │  │  观看展览   │     │
│    │  按钮       │  │  按钮       │     │
│    └────────────┘  └────────────┘     │
│                                         │
└─────────────────────────────────────────┘
         │                    │
         ▼                    ▼
┌─────────────────┐  ┌─────────────────┐
│  模块一：上传     │  │  模块二：观看     │
│  upload.html    │  │  gallery.html  │
└─────────────────┘  └─────────────────┘
```

---

## 二、主页布局 (index.html)

### 2.1 设计理念
- **简洁明了**：作为系统入口，只提供两个核心功能入口
- **视觉突出**：两大按钮要醒目，引导用户选择
- **响应式**：适配手机、平板、桌面

### 2.2 布局结构

```
┌─────────────────────────────────────┐
│         顶部标题/Logo                │
│      "云展览平台"                    │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│                                     │
│                                     │
│      ┌──────────────────┐           │
│      │                  │           │
│      │   上传照片        │           │
│      │   📤              │           │
│      │                  │           │
│      └──────────────────┘           │
│                                     │
│      ┌──────────────────┐           │
│      │                  │           │
│      │   观看云展览      │           │
│      │   🖼️              │           │
│      │                  │           │
│      └──────────────────┘           │
│                                     │
└─────────────────────────────────────┘
```

### 2.3 功能说明
- **上传照片按钮**：点击进入上传页面 (upload.html)
- **观看展览按钮**：点击进入展览页面 (gallery.html)
- **底部信息**：可显示已上传照片数量（可选）

## 三、模块一：上传照片页面 (upload.html)

### 3.1 设计理念
- **专注上传**：页面专注于照片上传功能
- **操作简单**：一键选择，实时预览
- **反馈及时**：上传进度、成功提示清晰

### 3.2 布局结构

```
┌─────────────────────────────────────┐
│  [返回主页]  上传照片                  │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│                                     │
│      ┌──────────────────┐           │
│      │                  │           │
│      │   点击选择照片    │           │
│      │   或拖拽到此处     │           │
│      │                  │           │
│      │   [选择文件按钮]  │           │
│      │                  │           │
│      └──────────────────┘           │
│                                     │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  预览区域（选择照片后显示）            │
│                                     │
│  已选择: 5/20 张照片                 │
│  ████████░░░░░░░░░░ 25%             │
│                                     │
│  [照片1] [照片2] [照片3] [照片4] [照片5]│
│                                     │
│  上传进度: ████████░░ 80%           │
│                                     │
│  [确认上传] [取消]                   │
└─────────────────────────────────────┘

注意：如果达到20张限制，会显示提示并禁用继续选择

┌─────────────────────────────────────┐
│  上传成功提示                         │
│  ✓ 已成功上传 X 张照片                │
│  [立即观看展览] [继续上传]             │
└─────────────────────────────────────┘
```

### 3.3 功能区域说明

#### 顶部导航区
- 返回主页按钮
- 页面标题"上传照片"

#### 上传选择区
- 大区域上传框（支持拖拽）
- 文件选择按钮（支持多选）
- 手机端：直接调用相簿选择

#### 预览区域
- 选择照片后实时显示缩略图
- 可删除已选择的照片
- 显示文件信息（名称、大小）
- **数量提示**：显示"已选择 X/20 张照片"
- **限制提示**：接近或达到20张时显示提示

#### 上传控制区
- 上传进度条
- 确认上传按钮
- 取消按钮
- **数量验证**：超过20张时禁用上传按钮并提示

#### 完成提示区
- 上传成功提示
- 跳转到展览页面按钮
- 继续上传按钮

## 四、模块二：观看云展览页面 (gallery.html)

### 4.1 设计理念
- **3D沉浸式体验**：核心功能，使用3D虚拟展览馆模拟实体展览馆
- **多种模式**：3D模式（核心）+ 传统模式（备选）
- **流畅浏览**：3D场景中自由"走动"，照片切换流畅

### 4.2 布局结构（3D展览馆模式 - 核心模式）

```
┌─────────────────────────────────────┐
│  [返回主页]  观看云展览  [3D/传统切换] │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│                                     │
│      ┌──────────────────────┐      │
│      │                      │      │
│      │                      │      │
│      │   3D展览馆场景        │      │
│      │   (WebGL Canvas)     │      │
│      │                      │      │
│      │  [照片挂在3D墙上]     │      │
│      │                      │      │
│      │  鼠标控制相机移动      │      │
│      │                      │      │
│      └──────────────────────┘      │
│                                     │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  控制面板（可选，可隐藏）              │
```

```
│    │         │                     │
```

- **移动端触摸**：单指拖拽旋转，双指缩放

### 4.5 功能区域说明

#### 顶部导航区
- 返回主页按钮
- 页面标题"观看云展览"
- **模式切换按钮**：3D模式 ↔ 传统模式

#### 3D展览展示区（核心）
- **WebGL Canvas**：全屏3D渲染区域
- **3D场景**：虚拟展览馆空间
- **照片挂墙**：照片以3D相框形式挂在虚拟墙上
- **相机控制**：用户通过鼠标/触摸控制视角
- **场景光照**：环境光和方向光营造真实感

#### 传统展览展示区（备选）
- **网格模式**：整齐的网格布局
- **瀑布流模式**：不同高度的照片流
- **全屏模式**：点击照片进入全屏浏览

#### 照片交互（3D模式）
- **鼠标悬停**：3D相框高亮显示
- **点击照片**：进入全屏/详情模式
- **相机移动**：在3D空间中"走动"浏览
- **射线检测**：精确检测点击的3D对象

#### 照片交互（传统模式）
- **点击照片**：进入全屏/详情模式
- **键盘导航**：方向键切换照片
- **移动端**：左右滑动切换

#### 控制功能
- **3D模式控制**：
  - 重置视角按钮
  - 帮助提示（操作说明）
  - 场景设置（光照、质量等）
- **传统模式控制**：
  - 上一张/下一张按钮
  - 缩略图导航
  - 照片信息显示（可选）

## 五、响应式布局设计

### 5.1 移动端布局 (< 768px)

#### 主页
- 两个按钮垂直排列
- 按钮占满屏幕宽度
- 大字体，易于点击

#### 上传页面
- 上传区域占满屏幕
- 预览照片单列显示
- 按钮大尺寸，易于触摸

#### 展览页面
- 照片单列或双列显示
- 全屏模式占满屏幕
- 触摸手势支持

### 5.2 平板布局 (768px - 1024px)

#### 主页
- 两个按钮并排或垂直排列
- 适中的按钮尺寸

#### 上传页面
- 预览照片2-3列显示
- 适中的上传区域

#### 展览页面
- 照片2-3列显示
- 混合交互（触摸+鼠标）

### 5.3 桌面布局 (> 1024px)

#### 主页
- 两个按钮并排显示
- 大尺寸，视觉突出

#### 上传页面
- 预览照片多列显示（3-4列）
- 侧边栏可显示上传历史（可选）

#### 展览页面
- 照片多列显示（3-4列）
- 完整的控制面板
- 鼠标悬停效果

## 六、页面间导航流程

```
主页 (index.html)
    │
    ├──► 点击"上传照片"
    │       │
    │       ▼
    │   上传页面 (upload.html)
    │       │
    │       ├──► 上传完成 → 跳转到展览页面
    │       └──► 返回主页
    │
    └──► 点击"观看展览"
            │
            ▼
        展览页面 (gallery.html)
            │
            ├──► 返回主页
            └──► 在展览页面内可删除照片
```

## 七、视觉设计要点

### 7.1 色彩方案
- **主色调**：选择一种主色（如蓝色、紫色）
- **背景色**：浅色背景（白色/浅灰）
- **强调色**：用于按钮和重要元素

### 7.2 字体
- **标题**：大号、粗体
- **正文**：清晰易读
- **按钮文字**：醒目

### 7.3 间距
- **充足留白**：避免拥挤
- **统一间距**：保持一致性
- **响应式间距**：不同屏幕适配

### 7.4 动画效果
- **页面切换**：平滑过渡
- **照片加载**：淡入效果
- **按钮交互**：悬停/点击反馈

## 八、布局总结

### 核心原则
1. **两大模块清晰分离**：上传和观看功能独立
2. **主页简洁明了**：只提供两个入口
3. **功能专注**：每个页面专注于一个核心功能
4. **流程顺畅**：页面间导航自然流畅

### 关键页面
- **index.html**：系统入口，选择功能
- **upload.html**：上传照片，完成存储
- **gallery.html**：观看展览，沉浸体验

### 用户体验
- **简单直观**：操作步骤少，易于理解
- **反馈及时**：每个操作都有明确反馈
- **流畅自然**：页面切换和交互流畅
